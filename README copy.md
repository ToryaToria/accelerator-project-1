## Описание

**Drink2Go** — это интернет-магазин по продаже кофейных напитков. <br>
Удобная сортировка по странам произрастания зерна и концентрации молока поможет выбрать идеальный напиток.<br>

## Техническое задание

### Требования к технологиям

- **SCSS + HTML** без шаблонизатора (использовать PUG или Twig в этом проекте нельзя);
- **БЭМ-методология** - рекомендована для использования;
- **готовая сборка** на основе GULP;
- **JavaScript (ES6+)**: - основной язык разработки, модульная архитектура;
- в этом проекте использовать нельзя:
    - JS-фреймворки (React или Vue);
    - CSS библиотеки типа Bootstrap;
    - leaflet и yandex-map;
- предусматривать работу без JavaScript не нужно.

### Адаптивность верстки

1. Используется подход **mobile-first**.

2. **Разрешения и точки перехода:**
  - минимальная ширина: 320px;
  - мобильный экран: от 320px;
  - планшетный экран: от 768px;
  - десктоп экран: от 1440px.

3. **Поведение между точками:**
  - для этого проекта важна скорость реализации, поэтому между точками перехода - только фиксированная ширина контента, без «резиновости»;
  - фоны и карта тянутся на всю ширину вьюпорта;

### Ретинизация и адаптивность изображений
  - адаптируем кадрирование логотипа, изображения в слайдере и карты;
  - ретинизируем изображение товара, изображения в слайдере и карты;
  - используем векторный спрайт для иконок: шапка, преимущества, кнопки, подвал;

### Интерактивность
1. Предусматривать работу без java script не нужно.

2. **Мобильное меню**:
    - видимость переключается тапом;
    - отображается поверх страницы, вне потока.

3. **Карта не интерактивная** - используются адаптивные изображения из макета;

4. **Переключение слайдов:**
  - реализовано без использования библиотек
  - переключение слайдов без зацикливания (навигация);
  - кнопки переключения слайдов блокированы в крайних положениях;
  - реализовать переход к произвольному слайду для десктопа (пагинация).

5. **Форма** (блок сортировки и фильтров):
  - выбор сортировки выполнен стандартным селектом, стилизация только внешняя;
  - кастомные чекбоксы и радиокнопки фильтров;
  - рейндж-слайдер для выбора цены реализуется с помощью библиотеки no-ui-slider
  - сортировку/фильтрацию/пагинацию списка товаров реализовывать не нужно, предполагается, что всё приходит с сервера

### Переполнение контентом:
  - Футер прижимается к нижней границе окна.
  - В меню шапки предусмотрите переменное число элементов.
  - В слайдах могут меняться: пропорции, размер изображений, количество и объём текста.
  - В блоке преимущества количество пунктов может измениться.
  - Количество контролов в фильтрах формы может измениться.
  - Число карточек в каталоге произвольное.
  - В меню подвала может меняться количество пунктов в колонках.
  - В элементе социальные сети — меняется количество ссылок.
